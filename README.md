# 🚀 Прокси-чекер (HTTPS)

Короче, эта штука нужна чтобы проверить список прокси на работоспособность. 
Берет файл, пингует каждый через HTTPS, меряет отклик и сохраняет только живые. 
Все просто, без лишней мишуры.

## 🔧 Что надо для запуска

1. **Python 3** (ну очевидно).
2. Библиотека `requests`. Если её нет, скрипт сам орет об этом при запуске, но лучше поставь заранее:
   ```bash
   pip install requests
   ```

## 🏃‍♂️ Как запустить

Закинь свои прокси в текстовый файл (по умолчанию скрипт ждет `proxies.txt`) и руби команду:

```bash
python Proxy_Checker.py
```

Или если файл называется по-другому:
```bash
python Proxy_Checker.py -f my_proxies.txt
```

## ⚙️ Настройки (аргументы)

Можно покрутить параметры, чтобы работало быстрее или медленнее, кому как надо:

| Аргумент | Что делает | По умолчанию |
|----------|------------|--------------|
| `-f`, `--file` | Путь к файлу с прокси | `proxies.txt` |
| `-o`, `--output` | Куда сохранить рабочие прокси | `working_proxies.txt` |
| `-t`, `--timeout` | Таймаут ожидания ответа (сек) | `10` |
| `-w`, `--workers` | Количество потоков (скорость) | `20` |
| `-u`, `--url` | Ссылка для проверки (куда стучаться) | `https://httpbin.org/ip` |

**Пример:**
```bash
python Proxy_Checker.py -f list.txt -w 50 -t 5
```
*Это запустит 50 потоков и будет ждать ответа максимум 5 секунд.*

## 📝 Формат прокси в файле

Скрипт не тупит, понимает большинство форматов (по одному на строку):
- `ip:port`
- `ip:port:username:password`
- `username:password@ip:port`
- `http://ip:port`
- `http://username:password@ip:port`

Комментарии можно оставлять через `#`, они игнорируются.

## 📊 Что на выходе

В консоль будет вываливаться результат в реальном времени:
- ✅ **Зеленые** — работают, показывается пинг и внешний IP.
- ❌ **Красные** — дохлые, показывается ошибка (таймаут, отказ и т.д.).

В конце будет итоговая табличка: сколько всего, сколько живых, средний пинг, лучший и худший прокси. 
Все рабочие сохранятся в отдельный файл и отсортируются по пингу (самые быстрые сверху).

## ⚠️ Нюансы

- Если файл пустой или не найден, скрипт честно об этом скажет (может даже грубо, но по делу).
- Скорость проверки зависит от количества потоков (`-w`). Не ставь слишком много, если интернет слабый.

1. Проверь, установлен ли `requests`.
2. Проверь, есть ли интернет (банально, но бывает).
3. Убедись, что в файле с прокси вообще что-то написано.
